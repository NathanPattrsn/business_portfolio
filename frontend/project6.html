React Native Wallet App – Personal Finance Tracker

The Wallet App is a cross-platform mobile and web application that allows users to manage their personal finances by tracking income and expenses. Users can view a dashboard summary, add new transactions, delete old ones, and securely log in/out using modern authentication. It currently isn't live but I can let you access it yourself upon request.

Purpose & Functionality

  • Dashboard Overview: Visualizes user's current financial state, showing total balance, income, and expenses.
  • Transaction Management: Users can add or delete transactions, with real-time updates reflected in the dashboard.
  • Authentication: Secure login and logout features using Clerk Authentication for user account handling.
  • Rate Limiting: Prevents abuse by limiting the number of API requests per user in a given time window.
  • API Communication: Frontend communicates with a hosted backend API to store and retrieve user-specific data.
  • Mobile & Web Compatibility:Runs on iOS, Android, and the web using the same codebase thanks to Expo and React Native Web.

Tools & Technologies Used

  • React Native with Expo – Built a cross-platform mobile-first UI compatible with both iOS and Android using Expo CLI for rapid development and testing.
  • React Native Web – Enabled web browser support using the same codebase, allowing seamless deployment for mobile and web.
  • React Navigation – Handled screen routing and user navigation flows across different views within the app.
  • Clerk – Integrated user authentication and session management across platforms with support for secure login and logout flows.
  • AsyncStorage – Used for local token storage and session persistence on mobile devices.
  • Node.js & Express.js – Created a REST API for handling CRUD operations on transactions and verifying user authentication.
  • Render – Hosted the backend server and deployed the API with automatic scaling and HTTPS support.
  • CORS Configuration – Managed cross-origin resource sharing to allow frontend access from web and mobile clients.
  • Middleware – Implemented features like rate limiting and authentication validation to improve backend performance and security.
  • JWT (JSON Web Tokens) – Enabled secure session management and protected API routes with user-specific access control.
  • EAS (Expo Application Services) – Used for building and deploying native binaries and enabling over-the-air (OTA) updates without store resubmission.
  • Over-the-Air (OTA) Updates – Deployed instant updates to users via Expo without needing app store approval cycles.
  • Sentry – Integrated for optional error tracking and monitoring in production environments for improved debugging and user experience.

What I Learned

  • Cross-Platform App Development – Built and deployed a production-ready mobile and web app from scratch using React Native with Expo and React Native Web.
  • Authentication Flows – Implemented user login, registration, and session management using Clerk across both web and native platforms.
  • RESTful API with Access Control – Developed a secure Node.js & Express backend with route protection using JWT authentication and custom middleware.
  • API Integration & CORS Management – Resolved complex CORS issues and ensured consistent API integration across mobile and web clients.
  • App Deployment & OTA Updates – Leveraged Expo EAS to build native app binaries and deliver instant over-the-air (OTA) updates without store delays.
  • State Management & Persistence – Managed application state and persisted user sessions using AsyncStorage and Clerk’s session APIs.

  • Modern Tools & Platforms – Utilized tools like Clerk (auth), Render (backend hosting), and Expo (cross-platform development) to streamline development and deployment.
  • Real-World Problem Solving – Addressed practical challenges including CORS issues, mobile vs. web compatibility, and production deployment strategies.
